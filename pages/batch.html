Infrastructure as Code · AWS Batch

두범(doobeom) Batch
스케줄링 인프라 프로젝트

Pulumi(Python) 기반 IaC로 AWS Batch 배치 작업 인프라를 선언적으로 관리합니다. EventBridge Scheduler, Step Functions, Lambda를 연동해 정기 배치의 실행·모니터링·알림을 자동화합니다.

회사 두범 (doobeom) 유형 Batch IaC 언어 Python / Pulumi 인프라 AWS · ap-northeast-2

01 프로젝트 개요

이커머스 플랫폼 두범(doobeom)의 백엔드 배치 작업(주문 처리, 포인트 만료, 랭킹 집계, 데이터 정책 등)을 AWS에서 안정적으로 스케줄링·실행하기 위한 Infrastructure as Code 프로젝트입니다.

06 멀티 환경 구성

Pulumi Stack으로 동일 코드베이스에서 환경별 리소스를 분리합니다.

환경 분리 (dev / stg / prod)

  • Profile doobeom-dev
  • ECR db-dev-batch
  • 스케줄 일부
  • VPC dev 전용
batch-stg
  • Profile doobeom-dev
  • ECR db-stg-batch
  • 스케줄 일부
  • VPC stg 전용
batch-prod
  • Profile doobeom-prod
  • ECR db-prod-batch
  • 스케줄 30+
  • VPC prod 격리
공통 코드베이스 · Stack Config만 환경별 YAML로 분기 · State: s3://doobeom-devops-iac-state/batch

Stack Config 예시

config:
  vpc:
    subnets: [subnet-xxx, subnet-yyy]
    security_groups: [sg-zzz]
  queue: [job-queue]
  definition:
    job-definition:
      image: "{ACCOUNT_ID}.dkr.ecr.ap-northeast-2.amazonaws.com/db-dev-batch:latest"
      cpu: 2
      memory: 4000
  eventbridge:
    schedules:
      - name: onlyDatabaseDestructionPolicy
        cron: "30 08 * * ? *"
        definition: job-definition
        queue: job-queue
        enabled: true

09 기술적 성과 및 역량

이 프로젝트에서 구현·운영한 핵심 역량입니다.

Infrastructure as Code

Pulumi Python SDK로 AWS 리소스를 코드화하고, S3 Remote Backend로 팀 State를 공유합니다.

이벤트 기반 오케스트레이션

EventBridge → Step Functions → Batch → Lambda 파이프라인으로 실행과 실패 알림을 자동화합니다.

멀티 환경 DevOps

Stack + YAML Config로 dev/stg/prod를 동일 코드에서 관리하고 VPC·ECR·스케줄을 분리합니다.

IAM 보안 설계

서비스별 Role·Policy 분리, AssumeRole·PassRole Condition으로 최소 권한을 적용합니다.

Observability

CloudWatch Logs(ALL), X-Ray(SFN/Lambda)로 실행 추적과 장애 분석을 지원합니다.

확장 가능한 스케줄링

YAML에 스케줄만 추가하면 프로비저닝되며, Definition 3종으로 리소스를 차등 할당합니다.
