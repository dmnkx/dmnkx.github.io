CB Analysis Importer
두범비즈팀 통계자료 업로더

CJ 출고 엑셀과 CCMC 원가 엑셀을 병합·정제해 로컬 xlsx로 저장하고, Google Sheets RAW 시트에 append하는 내부용 데스크톱 ETL 앱입니다. 분석팀이 반복하던 수작업 병합을 GUI 한 번의 업로드로 자동화합니다.

회사 두범 (doobeom) 패키지 cb-analysis-importer 패턴 MVVM 배포 Windows Installer → S3

1. Overview

유형데스크톱 ETL
출력Local xlsx + Sheets RAW
GUIPyQt5 + Fusion QSS
인증Service Account

2. 문제 정의와 해결

물류(출고)와 원가(마스터)가 서로 다른 엑셀로 흩어져, 채널별 매출·원가 분석을 매번 수작업으로 붙여야 했습니다.

Before VLOOKUP·피벗 수동 반복 · 채널 alias/취소 규칙을 사람이 기억 · 붙여넣기 오타·누락
After 두 파일 첨부 후 업로드 · convert() 정규화·집계·LEFT JOIN · ~/doobeombiz 저장 + Sheets append

6. ETL 파이프라인

핵심은 models/convert.py의 convert(fileInfo)입니다. 두 엑셀을 읽어 비즈니스 규칙을 적용한 뒤 분석용 정규 테이블을 반환합니다.

변환 단계
flowchart TD
  A["pd.read_excel CJ"] --> B["pd.read_excel CCMC\nskiprows=3"]
  B --> C["컬럼 rename"]
  C --> D["제품분류 leaf 추출"]
  D --> E["채널명 결정"]
  E --> F["channel_name_mapping"]
  F --> G["주문상태 != 주문취소"]
  G --> H["groupby + 수량 sum"]
  H --> I["LEFT JOIN 품목코드"]
  I --> J["inf/NaN → 0"]
  J --> K["구분 컬럼 · 날짜 포맷"]
  K --> L["result_df 반환"]
  1. 엑셀 로드 — CCMC는 skiprows=3
  2. 스키마 정규화 — 주문수량→수량, 금액→총 매출액(v+) 등
  3. 제품분류 leaf — "식품>냉동>만두" → 마지막 토큰
  4. 채널 결정 — SELF면 판매채널, 이후 alias dict
  5. 취소 건 제외 — 주문상태 != "주문취소"
  6. groupby 수량 합산
  7. 원가 LEFT JOIN — 품목코드 ↔ 제품코드

채널 정규화

스레드 분리
flowchart TB
  subgraph UIThread["Main / UI Thread"]
    FB1["FileButton CJ"]
    FB2["FileButton CCMC"]
    BTN["업로드 버튼"]
    LOAD["LoadingDialog"]
    MSG["QMessageBox"]
  end
  subgraph WorkerThread["UploadWorker"]
    RUN["run()"]
    APPEND["append_rows(df)"]
  end
  FB1 --> BTN
  FB2 --> BTN
  BTN -->|"동기: convert + to_excel"| DF["DataFrame"]
  DF --> LOAD
  DF --> WorkerThread
  RUN --> APPEND
  APPEND -->|"finished(bool,str)"| MSG

PyInstaller frozen 실행 시에는 sys._MEIPASS 아래 번들 경로를 사용합니다. 

def resource_path(relative_path):
    if hasattr(sys, "_MEIPASS"):
        return os.path.join(sys._MEIPASS, relative_path)
    return os.path.join(os.path.abspath("."), relative_path)

8. Google Sheets 연동

10. 기술 결정과 패턴

#결정이유
1MVVM 분리GUI 교체·CLI 재사용 시 비즈니스 로직 유지
2LEFT JOIN 원가원가 누락에도 매출 행 유지
3채널 alias dict원천 표기를 분석 표준명으로 흡수
4QThread 업로드Sheets 지연이 UI freeze로 이어지지 않음
5서비스 계정팀원별 OAuth·토큰 만료 제거
6로컬 xlsx + Sheets오프라인 검증본과 공유 원천 동시 확보
7release/debug 이중 빌드운영은 콘솔 숨김, 장애 대응은 로그
8OIDC → S3장기 AWS 키를 Actions에 넣지 않음
