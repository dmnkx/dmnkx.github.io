Ani Review — 애니 탐색 · 리뷰 커뮤니티

MyAnimeList 메타데이터는 Jikan API로 실시간 조회하고, 사용자 리뷰만 Supabase에 저장하는 정적 웹 앱입니다. Next.js Static Export + Vercel로 서버 없이 배포하는 풀스택 구조를 목표로 했습니다.

AI 활용 프로젝트 — 기획·구현·배포 과정에서 AI를 활용했으며, 개발에 걸린 시간은 약 30분입니다.

AI Assisted ~30분 Next.js 15 React 19 TypeScript Tailwind CSS 4 Jikan API Supabase Vercel

1. Overview

유형Static Web · Full-stack
역할기획 · 설계 · 개발 · 배포 (AI 활용)
개발 시간약 30분

수만 편의 애니 메타데이터를 직접 DB에 넣는 대신, 외부 API로 읽고 서비스에 필요한 리뷰만 DB에 저장하는 하이브리드 구조를 선택했습니다.

5. 핵심 설계 결정

Static Export output: "export" — 서버 없이 CDN 서빙, 데이터는 클라이언트 fetch
mal_id 단일 키 애니 테이블 제거 후 MAL ID로 Jikan·리뷰를 연결
동적 라우트 /anime/view?id= + vercel rewrite — export에서 [id] 사전 생성 불가 대응
레이트 리밋 검색 500ms 디바운스 · 요청 취소 · 실패 시 graceful degradation

6. 데이터베이스

ERD · reviews only
erDiagram
  reviews {
    uuid id PK
    integer mal_id
    text anime_title
    text anime_image
    text author_name
    integer rating
    text title
    text content
    timestamptz created_at
  }

anime 테이블 없음 — 애니 메타데이터의 Single Source of Truth는 Jikan입니다. RLS는 MVP에서 SELECT/INSERT 공개, 이후 Auth 연동 시 authenticated로 전환 가능하도록 설계했습니다.

인덱스용도
reviews_mal_id_idx애니 상세 리뷰 조회
reviews_created_at_idx최신 리뷰 정렬
