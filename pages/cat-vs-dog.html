5. 학습 유틸

노트북 초반에 반복 실험을 위한화하는 헬퍼를 둡니다.

get_callbacks()Checkpoint · EarlyStopping · ReduceLROnPlateau
get_model()Model 생성 · summary · compile
run()fit → predict → save_preds 일괄 실행
save_preds()샘플 제출 포맷에 맞춰 CSV 저장
def get_callbacks(early_patience=10, reduce_patience=3):
    checkpoint_cb = ModelCheckpoint(f"model/best-model_{get_filename()}.h5",
                                    save_best_only=True)
    early_stopping_cb = EarlyStopping(patience=early_patience,
                                      restore_best_weights=True)
    reduceLR_cb = ReduceLROnPlateau(monitor="val_loss",
                                    factor=0.5, patience=reduce_patience)
    return [checkpoint_cb, early_stopping_cb, reduceLR_cb]

7. 최종 아키텍처 (요지)

성능이 좋았던 계열은 Functional API로 서로 다른 kernel size의 Conv 분기를 concatenate한 뒤 추가 Conv·Dense로 이진 확률을 낸 구조입니다. 후반부에서는 ReLU 대신 Leaky ReLU를 사용했습니다.

Multi-kernel CNN
flowchart TB
  IN["Input 128x128x3"] --> C0["Conv2D 64 k=3"]
  C0 --> P0["MaxPool · Dropout"]
  P0 --> K4["Conv k=4"]
  P0 --> K5["Conv k=5"]
  P0 --> K6["Conv k=6"]
  K4 --> CAT["concatenate"]
  K5 --> CAT
  K6 --> CAT
  CAT --> C2["Conv · Pool · Dropout × N"]
  C2 --> FL["Flatten"]
  FL --> D1["Dense · Dropout"]
  D1 --> OUT["Dense 1 · Sigmoid"]
          
x1 = Conv2D(64, 4, padding="same", activation=tf.nn.leaky_relu)(x)
x2 = Conv2D(64, 5, padding="same", activation=tf.nn.leaky_relu)(x)
x3 = Conv2D(64, 6, padding="same", activation=tf.nn.leaky_relu)(x)
x  = concatenate([x1, x2, x3])
# … 추가 Conv / Dense …
outputs = Dense(1, activation=tf.nn.sigmoid)(x)
